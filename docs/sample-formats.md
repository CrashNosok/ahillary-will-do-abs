# Форматы сэмплов (для парсеров)

Назначение — зафиксировать факты о входных файлах, чтобы парсеры не пришлось
переоткрывать заново. Всё ниже выведено из реальных файлов в `samples/`:
`FoodDiary_260620_foods.csv` (FatSecret) и `IMG_9605.PNG` (Welltory).

> Примеры значений — это конкретные показания из приложенных сэмплов, а не
> инварианты. Парсер не должен закладываться на сами числа, только на форматы.

## Сопоставление файлов (glob)

| Источник | Glob | Пример из `samples/` |
| --- | --- | --- |
| FatSecret — экспорт дневника питания | `FoodDiary_*_foods.csv` | `FoodDiary_260620_foods.csv` |
| Welltory — скрин экрана активности | `IMG_*.PNG` | `IMG_9605.PNG` |

- Расширение `.PNG` — **в верхнем регистре** (как экспортит iOS). Матчинг по
  glob должен учитывать регистр или матчить регистронезависимо.
- В имени `FoodDiary_<YYMMDD>_foods.csv` дата — `YYMMDD`: `260620` = 2026-06-20
  (совпадает с датой внутри отчёта «суббота, июня 20, 2026»). Это дата экспорта/
  отчёта; полагаться на содержимое, а не парсить дату из имени.

## 1. FatSecret CSV — `FoodDiary_*_foods.csv`

### Кодировка и переводы строк

- **BOM:** `EF BB BF` в начале файла (UTF-8 with BOM). Читать как `utf-8-sig`,
  иначе первый `#` склеится с BOM.
- **Переводы строк:** `CRLF` (`\r\n`).

### Строки-комментарии и пустые строки

- Любая строка, начинающаяся с `#`, — комментарий. В сэмпле это баннеры
  (`#-------…`), `# Food Diary Report - Detailed Report,"суббота, июня 20, 2026"`
  и `# Report Details`.
- **ВАЖНО:** проверять префикс `#` на сырой строке **до** CSV-парсинга — строки-
  комментарии сами содержат запятые и закавыченные поля, наивный CSV-ридер их
  не отличит от данных.
- Пустые строки (между секциями и перед `Всего`) пропускать.
- Строка-заголовок колонок (`Дата,Кал ( ккал),…`) — **не** комментарий (без `#`).

### Колонки — маппинг по индексу (0-based)

Строка-заголовок:
`Дата,Кал ( ккал),Жир( г),Н/жир( г),Углев( г),Клетч( г),Сахар( г),Белк( г),Натри( мг),Холес( мг),Калий( мг)`

| Индекс | Заголовок | Поле | Берём |
| --- | --- | --- | --- |
| 0 | Дата | название строки (дата/приём/продукт/порция) | — |
| 1 | Кал ( ккал) | `kcal` | **да** |
| 2 | Жир( г) | `fat` | **да** |
| 3 | Н/жир( г) | насыщенные жиры | нет |
| 4 | Углев( г) | `carbs` | **да** |
| 5 | Клетч( г) | клетчатка | нет |
| 6 | Сахар( г) | сахар | нет |
| 7 | Белк( г) | `protein` | **да** |
| 8 | Натри( мг) | натрий | нет |
| 9 | Холес( мг) | холестерин | нет |
| 10 | Калий( мг) | калий | нет |

Маппинг для парсера: **kcal=1, fat=2, carbs=4, protein=7**.
Матчить по позиции (индексу), **не** по тексту заголовка — заголовки русские,
с нестабильными пробелами (`Кал ( ккал)` vs `Жир( г)`) и единицами в скобках.

### Десятичные, пустые ячейки, дата

- **Десятичные** — запятая-разделитель, поле в кавычках: `"170,4"`, `"138,45"`,
  `"6,6"`. Преобразование: убрать кавычки, `,` → `.`.
- **Целые** — без кавычек и без разделителя тысяч: `3250`, `455`, `0`.
  (Кавычки появляются только когда внутри поля есть запятая.)
- **Пустая ячейка** = нет данных: подряд идущие запятые `,,` (в сэмпле пусты
  Н/жир, Клетч, Сахар, Натри, Холес, Калий).
- **RU-дата** — в кавычках, т.к. содержит запятые: `"суббота, июня 20, 2026"`.
  Формат: `<день недели>, <месяц в род. падеже> <число>, <год>`, день недели в
  нижнем регистре, месяц в родительном падеже («июня», не «июнь»).

### Иерархия по ведущим пробелам

Уровень вложенности = число **ведущих пробелов** (литеральные `0x20`, не табы) в
первой ячейке:

| Пробелов | Уровень | Что это | Несёт нутриенты |
| --- | --- | --- | --- |
| 0 | день / Всего | строка-дата (первая ячейка — дата в кавычках) или grand-total (первая ячейка — `Всего`) | да (дневной итог) |
| 1 | приём пищи | Завтрак / Обед / Ужин / Перекус/Другое | да (итог приёма) |
| 2 | продукт | конкретный продукт | да (за съеденную порцию) |
| 3 | порция | строка только с размером порции («300 г») | нет — лишь текст порции |

- Строка уровня 3 (порция) идёт **сразу под** своим продуктом, привязывается к
  нему; колонок-значений у неё нет, только `   <число> <единица>` (в сэмпле `г`).
- На уровне 0 две разновидности: строка-дата (дневной итог) и `Всего` (общий
  итог) — они **дублируют** друг друга; не суммировать дважды, выбрать одну.

### Краевые случаи (чеклист парсера)

- [ ] BOM `EF BB BF` снят (`utf-8-sig`).
- [ ] `CRLF` нормализован.
- [ ] Строки на `#` отфильтрованы **до** CSV-парсинга (внутри них есть `,` и `"`).
- [ ] Глубина считается по ведущим пробелам (0/1/2/3), а не по содержимому.
- [ ] Десятичные в кавычках с запятой (`"6,6"`) → `6.6`; целые без кавычек.
- [ ] Пустые ячейки `,,` → None/0 по смыслу поля.
- [ ] Приём без продуктов: ` Перекус/Другое,,,,,,,,,,` — все нутриенты пусты.
- [ ] Имя продукта со спецсимволами: апостроф и дефис
      (`Ростик'c - KFC Крылышки Острые`).
- [ ] `Всего` дублирует строку-дату — не учитывать обе.
- [ ] Колонки матчатся по индексу, не по русскому заголовку.

## 2. Welltory — скрин активности `IMG_*.PNG`

Источник — iOS-скриншот экрана **«Анализ активности» → вкладка «Дни» →
выбранный день**. Это растровое изображение, не структурированный экспорт:
извлечение через OCR/ручной ввод.

Дата выбранного дня видна на самом скрине как «Сб, 20 июня» (краткий день недели)
— тот же день, что и в FatSecret-сэмпле, но формат другой.

### Поля (плитки в нижней части экрана)

| Ключ | Ярлык на скрине | Пример | Единица / формат |
| --- | --- | --- | --- |
| `всего_ккал` | ВСЕГО ККАЛ | 1218 ккал | целое + « ккал» |
| `активные_ккал` | АКТ. ККАЛ | 683 ккал | целое + « ккал» |
| `шаги` | ШАГИ | 4459 | целое, без единицы |
| `в_движении` | В ДВИЖЕНИИ | 2ч 53м | длительность |
| `без_движения` | БЕЗ ДВИЖ. | 21ч 57м | длительность |
| `разминка` | РАЗМИНКА | 7ч | длительность |
| `активные_мет` | АКТ. МЕТ | 782 МЕТ | целое + « МЕТ» |
| `интенсивные_мет` | ИНТ. МЕТ | 0 МЕТ | целое + « МЕТ» |

На скрине есть ещё одна плитка, не вошедшая в исходный список карточки, но
присутствующая в сетке: **ИНТ. АКТИВ.** = `0 мин` (длительность интенсивной
активности). Парсер встретит 9 плиток, не 8.

### Форматы значений

- **ккал / МЕТ:** целое, пробел, единица — `1218 ккал`, `782 МЕТ`.
- **шаги:** целое без единицы — `4459`.
- **Длительности неконсистентны** — парсер должен принять все три формы:
  - `2ч 53м` — часы + минуты, сокращённо `ч`/`м`;
  - `7ч` — минуты опущены, когда они нулевые;
  - `0 мин` — суб-часовое/нулевое значение пишется полным «мин».
- В правом верхнем углу выбранного дня дублируются `в_движении` и
  `активные_мет` (`2ч 53м`, `782 МЕТ`).

### Краевые случаи (чеклист)

- [ ] Растр → OCR/ручной ввод; структурированных полей нет.
- [ ] 9 плиток (включая ИНТ. АКТИВ.), а не 8 из списка карточки.
- [ ] Длительность в трёх формах (`Хч Ym` / `Хч` / `N мин`).
- [ ] Единицы прилипают к числу через пробел; шаги — без единицы.

## Проверка (smoke)

Факты выше воспроизводятся из сэмплов:

```bash
# BOM + CRLF + кодировка
file samples/FoodDiary_260620_foods.csv
# → UTF-8 (with BOM) text, with CRLF line terminators

# Структура с уровнями вложенности (ведущие пробелы).
# read_bytes(), а не read_text() — text-режим съедает CRLF и split не сработает.
python3 -c "import pathlib; t=pathlib.Path('samples/FoodDiary_260620_foods.csv').read_bytes().decode('utf-8-sig'); [print(len(l)-len(l.lstrip(' ')), repr(l)) for l in t.split(chr(13)+chr(10))]"
```

Welltory-поля сверяются визуально по `samples/IMG_9605.PNG`.
