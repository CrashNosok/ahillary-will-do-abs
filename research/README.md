# Корпус исследований (research corpus)

Подборка реальных научных работ по составу тела и тренировкам — основа для **доказательного
«Плана от модели»** (отчёт ссылается на эти исследования) и задел под будущий RAG.

## Что здесь

| Путь | Коммитим? | Что это |
|---|---|---|
| `studies.json` | да | Индекс-метаданные (источник правды): id, заголовок, авторы, год, журнал, DOI/URL, темы, тип, аннотация, путь к PDF |
| `abstracts/<id>.txt` | да | Аннотации в чистом тексте (для отчёта и будущего RAG-чанкинга) |
| `pdfs/<id>.pdf` | да | Полные тексты (Open Access). По решению проекта коммитим вместе с индексом |

`id` — slug из DOI (например `10-3390-nu13041174`). Один `id` стабилен между прогонами.

## Как (пере)собрать

```bash
backend/.venv/bin/python scripts/research/fetch_studies.py --email you@example.com
# одна тема:        --topic creatine
# больше на тему:   --per-topic 6
# только план:      --dry-run
# без скачивания:   --no-pdf
```

Источник — **OpenAlex** REST (без API-ключа; OA-PDF от Unpaywall). На каждую тему делается
релевантный поиск по заголовку+аннотации с фильтром Open Access, сортировка по цитируемости
(влиятельные мета-анализы/RCT/обзоры наверху). PDF берётся из OA-локаций (перебираем все,
оставляем тот URL, что реально отдаёт `%PDF`). Скрипт **идемпотентен**: индекс upsert по `id`,
уже скачанные PDF не перекачиваются, темы объединяются.

Темы покрывают все три типа цели (набор массы / сушка / поддержание) плюс сопутствующую
науку (углеводы/клетчатка, креатин, сон/восстановление) — см. словарь `TOPICS` в скрипте.

## Где используется

- `backend/app/services/research.py` грузит `studies.json`, строит «evidence-pack» для промпта
  и множество валидных id цитат (отчёт может ссылаться **только** на работы из корпуса —
  выдуманные ссылки отбраковываются).
- Отчёт цитирует работы инлайн как `[<id>]` и отдаёт список ссылок с DOI/URL.

> Исследования — англоязычные (наука по теме в основном на английском); отчёт пишется
> по-русски и ссылается на оригиналы по DOI.
